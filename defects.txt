МЕТОД:               capitalize()
Местоположение:      string_utils.py, класс StringUtils

ЗАГОЛОВОК:
Метод capitalize() изменяет регистр всех букв, а не только первой

ОПИСАНИЕ:
Метод использует встроенный str.capitalize(), который не только делает
первую букву заглавной, но и приводит все остальные буквы к нижнему регистру.
Это противоречит документации метода, где сказано:
"делает первую букву заглавной и возвращает этот же текст".

ОЖИДАЕМОЕ ПОВЕДЕНИЕ:
capitalize("SkyPro") -> "SkyPro"
capitalize("TEST")   -> "TEST"

ФАКТИЧЕСКОЕ ПОВЕДЕНИЕ:
capitalize("SkyPro") -> "Skypro"
capitalize("TEST")   -> "Test"

ШАГИ ДЛЯ ВОСПРОИЗВЕДЕНИЯ:
1. Создать экземпляр класса StringUtils
2. Вызвать метод capitalize() с аргументом "SkyPro"
3. Сравнить результат с ожидаемым "SkyPro"

ПРЕДЛАГАЕМОЕ ИСПРАВЛЕНИЕ:
Вместо string.capitalize() использовать:
    return string[0].upper() + string[1:] if string else string

ЗАТРАГИВАЕТ: Все вызовы метода capitalize(), где строка содержит
заглавные буквы не только в начале.

МЕТОД:               trim()
Местоположение:      string_utils.py, класс StringUtils

ЗАГОЛОВОК:
Метод trim() не удаляет другие пробельные символы (табуляцию, перенос строки)

ОПИСАНИЕ:
Метод удаляет только символ пробела (" ") в начале строки.
Другие пробельные символы, такие как табуляция (\t) и перенос строки (\n),
не удаляются. Это может быть неочевидным для пользователей, ожидающих
поведения, аналогичного методу strip().

ОЖИДАЕМОЕ ПОВЕДЕНИЕ (исходя из названия метода):
trim("\t  skypro") -> "skypro"

ФАКТИЧЕСКОЕ ПОВЕДЕНИЕ:
trim("\t  skypro") -> "\t  skypro"

ШАГИ ДЛЯ ВОСПРОИЗВЕДЕНИЯ:
1. Создать экземпляр класса StringUtils
2. Вызвать метод trim() с аргументом "\t  skypro"
3. Убедиться, что табуляция не удалена

ПРИМЕЧАНИЕ:
Может являться не дефектом, а особенностью реализации, если метод
действительно должен удалять только пробелы. Рекомендуется уточнить
требования или переименовать метод (например, в remove_leading_spaces).

МЕТОД:               contains()
Местоположение:      string_utils.py, класс StringUtils

ЗАГОЛОВОК:
Метод contains() возвращает True для пустого искомого символа

ОПИСАНИЕ:
При передаче пустой строки в качестве искомого символа метод возвращает True,
так как string.index("") возвращает 0, и условие 0 > -1 истинно.
Это может вводить в заблуждение при фильтрации или валидации данных.

ОЖИДАЕМОЕ ПОВЕДЕНИЕ (один из вариантов):
- Возвращать False (пустой символ не считается искомым)
- Выбрасывать исключение ValueError с понятным сообщением

ФАКТИЧЕСКОЕ ПОВЕДЕНИЕ:
contains("SkyPro", "") -> True
contains("", "")       -> True

ШАГИ ДЛЯ ВОСПРОИЗВЕДЕНИЯ:
1. Создать экземпляр класса StringUtils
2. Вызвать метод contains() с аргументами ("SkyPro", "")

ПРЕДЛАГАЕМОЕ ИСПРАВЛЕНИЕ:
Добавить проверку в начало метода:
    if not symbol:
        raise ValueError("Искомый символ не может быть пустым")

МЕТОД:               delete_symbol()
Местоположение:      string_utils.py, класс StringUtils

ЗАГОЛОВОК:
Избыточный вызов contains() в методе delete_symbol()

ОПИСАНИЕ:
Метод delete_symbol() вызывает self.contains() перед заменой.
Это избыточно, так как str.replace() корректно отрабатывает,
даже если искомая подстрока отсутствует (возвращает исходную строку).
Двойной поиск подстроки (сначала в contains, затем в replace)
снижает производительность.

ОЖИДАЕМОЕ ПОВЕДЕНИЕ:
delete_symbol("SkyPro", "X") -> "SkyPro"  (работает и так)

ШАГИ ДЛЯ ВОСПРОИЗВЕДЕНИЯ:
1. Вызвать delete_symbol("SkyPro", "X")
2. Убедиться, что результат корректен
3. Проверить, что contains() вызывается без необходимости

ПРЕДЛАГАЕМОЕ ИСПРАВЛЕНИЕ:
Упростить метод до:
    def delete_symbol(self, string: str, symbol: str) -> str:
        return string.replace(symbol, "")

ПРИМЕЧАНИЕ: Не влияет на корректность, только на производительность
и читаемость кода.
